## 优化策略

### 1. 使用CPU版本PyTorch

| 版本 | 打包后大小 | 说明 |
|------|-----------|------|
| GPU版本 | ~4 GB | 包含CUDA库，但你的电脑可能无法使用 |
| CPU版本 | ~1 GB | 体积小，功能完全相同（无GPU） |

**建议**：如果你的电脑没有NVIDIA显卡，使用CPU版本。

### 2. 排除静态库文件(.lib) - 核心优化

**使用 Tree 的 excludes 参数**

在 `build_working.spec` 中，通过 `Tree` 函数排除 `.lib` 文件：

```python
from PyInstaller.building.build_main import Tree

# 获取库路径
def get_numpy_path():
    import numpy
    return numpy.__path__[0]

def get_pandas_path():
    import pandas
    return pandas.__path__[0]

def get_torch_path():
    import torch
    return torch.__path__[0]

# 使用Tree确保关键目录完整，但排除.lib静态库
# 排除静态库文件(.lib)以减小体积，这些文件编译时需要但运行时不需要
numpy_tree = Tree(get_numpy_path(), prefix='numpy', excludes=["*.pyc", "__pycache__", "*.lib"])
pandas_tree = Tree(get_pandas_path(), prefix='pandas', excludes=["*.pyc", "__pycache__", "*.lib"])
torch_tree = Tree(get_torch_path(), prefix='torch', excludes=["*.pyc", "__pycache__", "test", "testing", "*.lib"])

# 添加到datas
a.datas += numpy_tree
a.datas += pandas_tree
a.datas += torch_tree
```

**效果**：减少约 **750MB**（从 1.8GB 降到 1GB）

**为什么用 Tree 的 excludes？**

| 方法 | 结果 | 原因 |
|------|------|------|
| 手动删除.lib文件 | ❌ 失败 | 误删其他文件导致NumPy导入错误 |
| Analysis的excludes | ❌ 无效 | Tree强制包含整个目录，excludes被覆盖 |
| **Tree的excludes** | ✅ **成功** | 在源头排除，不影响运行时 |

**原理**：
- `.lib` 是静态链接库，编译C/C++程序时需要
- `.dll` 是动态链接库，Python程序运行时只需要 `.dll`
- 通过 `Tree` 的 `excludes` 参数在打包时排除 `.lib`，既减小体积又不影响功能

## 常见问题

### Q: 打包后程序无法运行？

A: 检查是否：
1. 使用了GPU版本PyTorch但电脑无NVIDIA显卡
2. 缺少必要的hiddenimports
3. 使用了 `--onefile` 模式（不稳定）

**解决**：使用 `build_working.spec` 中的配置，使用 `COLLECT` 模式。

### Q: 打包体积还是太大？

A: 1GB是PyTorch程序的合理体积，包含：
- PyTorch核心 (~200MB)
- Intel MKL数学库 (~400MB，必须)
- 其他依赖 (~400MB)

如需更小体积，考虑：
- 使用更小的替代库（如用numpy替代pandas）
- 不打包，提供源码让用户自行安装

### Q: 如何更新打包？

A: 修改代码后重新执行：

```cmd
rmdir /s /q build dist
pyinstaller build_working.spec
```
