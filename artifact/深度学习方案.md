# 圣遗物预测器深度学习方案

## 一、项目概述

### 1.1 目标
- 收集用户强化记录数据
- 使用深度学习（PyTorch）训练预测模型
- 提供统计模式 + 深度学习模式双预测方案

### 1.2 技术栈
- **PyQt5**: GUI界面
- **PyTorch 2.0+**: 深度学习框架
- **JSON**: 主数据存储格式
- **Pandas/OpenPyXL**: Excel导出功能

## 二、数据存储方案

### 2.1 文件结构
```
强化记录/
├── record_YYYYMMDD_HHMMSS.json  # 单次强化记录（每次保存生成独立文件）
├── models/
│   ├── best_model.pt            # 最新模型（默认加载）
│   ├── model_YYYYMMDD_HHMMSS.pt # 历史版本（带时间戳）
│   └── training_log.txt         # 训练日志
└── exports/                     # 导出文件夹
    └── records_YYYYMMDD_HHMMSS.xlsx  # 导出的历史数据Excel
```

**说明**：
- **数据记录**：每次点击"保存数据"生成一个独立的JSON文件（精确到秒，避免覆盖）
- **模型版本**：每次训练会保存两个文件：
  - `model_YYYYMMDD_HHMMSS.pt`：带时间戳的历史版本（永久保留）
  - `best_model.pt`：最新模型（用于预测，会被覆盖）
- 可以手动切换使用历史版本模型进行预测

### 2.2 JSON数据结构

每个JSON文件包含一条强化记录：

```json
{
  "records": [
    {
      "timestamp": "2024-01-15 10:30:00",
      "entries": [
        {"value": "f16", "gear": 1, "star": "5星", "type": "f", "order": 1},
        {"value": "s239", "gear": 2, "star": "3星", "type": "s", "order": 2},
        {"value": "g18", "gear": 3, "star": "3星", "type": "g", "order": 3},
        {"value": "F73", "gear": 4, "star": "5星", "type": "F", "order": 4}
      ],
      "artifact_type": "羽毛"
    }
  ],
  "metadata": {
    "total_records": 1,
    "last_updated": "2024-01-15 10:30:00"
  }
}
```

**说明**：
- 虽然结构支持多条记录，但实际每个文件只存一条
- 方便后续扩展，也保持与训练数据加载逻辑一致

### 2.3 字段说明

## 三、深度学习模型设计

### 3.1 模型架构
```
输入层: 3个词条的编码 + skip参数
    ↓
Embedding层: 80种组合 → 16维向量
    ↓
LSTM层: 隐藏层64，2层，dropout=0.2
    ↓
Skip嵌入层: skip(1-MAX_SKIP) → 4维向量
    ↓
拼接: LSTM输出 + Skip嵌入
    ↓
共享全连接层: (64+4) → 32
    ↓
├─ 类型预测头: 32 → 10 (Softmax) → 词条类型概率
└─ 档位预测头: 32 → 4 (Softmax) → 档位概率(1-4档)
```

**多任务学习设计**：
- **共享特征提取**：Embedding + LSTM + 共享FC层同时服务于类型和档位预测
- **独立预测头**：类型预测头(10类)和档位预测头(4类)分别输出
- **联合训练**：总损失 = 类型损失 + 档位损失
- **同时预测**：一次前向传播，同时获得类型和档位概率

**各层次解释：**
- **输入层**：接收3个词条的数字编码（如[40, 5, 14]）和skip参数（1-3）
- **Embedding层**：把0-79的编码变成16维向量，让模型理解词条间的关系（比如f和F都是防御相关）
- **LSTM层**：分析序列规律，记住前3个词条的信息
- **Skip嵌入层**：把skip参数（预测步长）编码成4维向量，告诉模型要预测多远
- **拼接层**：把LSTM输出和Skip嵌入拼接，综合序列信息和预测距离
- **全连接层**：整合信息，提取关键特征
- **输出层**：输出10种词条类型的概率分布

**Skip参数的作用**：
- 让模型知道是预测下一个（skip=1）、下下个（skip=2）还是下下下个（skip=3）
- 同一个3元组，skip不同，预测目标不同
- 例如：[A,B,C] + skip=1 → 预测D；[A,B,C] + skip=2 → 预测E

### 3.2 特征编码

**为什么要编码？**  
计算机只认识数字，所以需要把词条转换成数字。

| 特征 | 取值范围 | 编码方式 |
|------|----------|----------|
| 词条类型 | 10种 | 0-9 |
| 档位 | 4档 | 0-3 |
| 星级 | 2种 | 3星=0, 5星=1 |
| **组合ID** | **80种** | **类型×4 + 档位 + 星级×40** |

**编码原理：**
```
组合ID = 类型×4 + 档位 + 星级×40
```
- 类型有10种（f,s,g,F,S,G,j,c,b,B）→ 0-9
- 每个类型有4个档位 → 0-3
- 星级2种（3星/5星）→ 0或1
- 组合ID范围：0-79（共80种组合）

编码示例:
- f16(5星,档位1): 类型f=0, 档位=0, 星级=1 → ID = 0×4 + 0 + 1×40 = **40**
- s239(3星,档位2): 类型s=1, 档位=1, 星级=0 → ID = 1×4 + 1 + 0×40 = **5**

### 3.3 训练样本构建

训练样本：就像教小孩认字，需要给模型看"例题"。

**基础样本（skip=1）**：
```python
# 一条强化记录（4个词条）
entries = [f16, s239, g18, F73]

# 滑动窗口构建多个训练样本
样本1: [f16, s239, g18] + skip=1 → 预测 F73
样本2: [s239, g18, F73] + skip=1 → 预测 下一个（如果有）

# 输入: ([40, 5, 14], skip=1) (3个组合ID + skip)
# 输出: 
#   - 类型标签: 3 (F类型的索引)
#   - 档位标签: 3 (4档，因为F73是4档)
```

**双标签训练**：
每个训练样本包含两个标签：
- `target_type`: 词条类型（0-9，对应10种类型）
- `target_gear`: 档位（0-3，对应1-4档）

模型同时学习类型预测和档位预测，共享特征提取层，通过独立输出头分别预测。

**Skip变体样本（支持长程预测）**：
对于序列 [A,B,C,D,E]，生成多种skip的训练样本：
```
输入[A,B,C] + skip=1 → 预测 D（近期）
输入[A,B,C] + skip=2 → 预测 E（中期）
输入[B,C,D] + skip=1 → 预测 E（近期）
输入[B,C,D] + skip=2 → 预测 F（如果有，长期）
```

**为什么用滑动窗口+Skip？**  
- 一条N词条的记录可生成多个训练样本
- 每个3元组可生成多个skip变体
- 例如6条词条 [A,B,C,D,E,F] 可生成：
  - skip=1: [A,B,C]→D, [B,C,D]→E, [C,D,E]→F
  - skip=2: [A,B,C]→E, [B,C,D]→F
  - skip=3: [A,B,C]→F
- 数据利用率更高，同时学习近期和长期模式
- **支持最大skip数**：由 `config.MAX_SKIP` 配置决定（默认4，可配置1-5）

**样本权重**（双重加权）：

1. **跨星级加权**：跨星级样本基础权重×2
   - 纯3星/纯5星：基础权重 = 1.0
   - 跨星级：基础权重 = 2.0（`STAR_TRANSITION_WEIGHT`）

2. **Skip折扣**：skip越大权重略低（近期预测更可靠）
   - skip=1：折扣 1.0（无折扣）
   - skip=2：折扣 0.9
   - skip=3：折扣 0.8
   - skip=4：折扣 0.7
   - 公式：`skip_discount = 1.0 - (skip-1) × 0.1`

3. **最终权重**：`weight = base_weight × skip_discount`

示例：
- 纯3星 + skip=1：1.0 × 1.0 = 1.0
- 跨星级 + skip=2：2.0 × 0.9 = 1.8
- 纯5星 + skip=3：1.0 × 0.8 = 0.8
- 纯5星 + skip=4：1.0 × 0.7 = 0.7

### 3.4 模型参数

| 参数 | 值 | 💡 解释 |
|------|-----|---------|
| 嵌入维度 | 16 | 每个词条编码变成16维向量，捕捉词条间关系 |
| LSTM隐藏层 | 64 | LSTM层有64个神经元，处理序列信息 |
| LSTM层数 | 2 | 2层LSTM堆叠，增强学习能力 |
| Dropout | 0.2 | 随机丢弃20%神经元，防止死记硬背（过拟合） |
| 学习率 | 0.001 | 每次参数调整的步伐大小，太大不稳定，太小学得慢 |
| Batch Size | 32 | 每次训练看32个样本，平衡速度和稳定性 |
| Epoch | 100（早停） | 最多训练100轮，如果没进步就提前停止 |
| 优化器 | Adam | 自适应学习率优化算法，自动调整学习步伐 |
| 损失函数 | CrossEntropyLoss | 分类问题常用，衡量预测和真实的差距 |
| 早停耐心值 | 10 | 连续10轮没有提升就停止训练 |
| 类型类别数 | 10 | 10种词条类型（f,s,g,F,S,G,j,c,b,B） |
| 档位类别数 | 4 | 4个档位（1档/2档/3档/4档） |

## 四、训练流程

### 4.1 触发条件
- **最小数据量**: 10条强化记录（可在config.py中调整）
- **触发方式**: 手动点击"训练模型"按钮

**为什么需要10条？**  
- <10条：数据太少，模型无法学习
- 10-50条：初步可用，但效果有限
- 100条+：基本可用（80种组合，每种至少出现1-2次）
- 500条+：比较准确

### 4.2 训练步骤

1. **读取所有record_*.json文件** - 收集所有强化记录作为"教材"
2. **构建训练样本（滑动窗口3→1）** - 把每条4词条记录变成1个训练样本
3. **划分训练集/测试集（80%/20%）** - 80%用来学习，20%用来考试检验学习效果
4. **训练模型** - 反复学习，不断调整参数
5. **评估测试集准确率** - 用20%的测试数据检验学得怎么样
6. **保存最佳模型（best_model.pt）** - 保存考得最好的那次
7. **记录训练日志** - 记录每次考试的分数，看进步情况

## 五、深度学习预测模式

### 5.1 模式选择

| 模式 | 说明 | 启用条件 |
|------|------|----------|
| 统计模式 | 使用genmat401.npy查表，通用统计 | 始终可用 |
| 深度学习模式 | 使用PyTorch模型推理，个性化预测 | 模型文件存在（best_model.pt）|

**两种模式的区别：**
- **统计模式**：用所有人的历史数据统计，通用但不个性化
- **深度学习模式**：用你自己的数据训练，个性化但需积累数据

### 5.2 预测流程

**核心：多skip策略预测同一目标**

```
录入: [A, B, C, D, E, F, G] (7个词条，预测G后面那个)

skip=1: [E, F, G] → 预测H (走1步，权重0.55)
skip=2: [D, E, F] → 预测H (走2步，权重0.25)
skip=3: [C, D, E] → 预测H (走3步，权重0.15)
skip=4: [B, C, D] → 预测H (走4步，权重0.05)

结果 = 0.55×skip1 + 0.25×skip2 + 0.15×skip3 + 0.05×skip4
```

**不同词条数的skip选择**：

| 词条数 | 可用skip | 说明 |
|--------|---------|------|
| <3 | 无 | 无法预测 |
| 3 | 1 | 只有skip=1 |
| 4 | 1, 2 | 近期+中期 |
| 5-6 | 1, 2, 3 | 近期+中期+长期 |
| ≥7 | 1, 2, 3, 4 | 四者综合 |

**权重配置**（config.py）：
- skip=1：0.55（近期最可靠，总和=1.0）
- skip=2：0.25（中期趋势）
- skip=3：0.15（长期规律）
- skip=4：0.05（超长期）

**动态权重**：
- 5-6词条时：skip=1 使用 `SKIP_1_WEIGHT_5TO6`（默认0.55）
- ≥7词条时：skip=1 使用 `SKIP_1_WEIGHT_7PLUS`（默认0.55）

**模型兼容性**：
- 旧模型（max_skip=3）与 skip=4 配置兼容
- 预测时自动检测模型支持的 max_skip，取配置和模型的最小值

### 5.3 预测辅助功能

#### Top-3推荐
在表1下方显示概率最高的3个词条，方便快速决策：
```
Top-3推荐: 1.大防(35.0%) | 2.充能(15.0%) | 3.精通(12.0%)
```

#### 档位预测
基于深度学习模型，同时预测下一个词条的档位概率：
```
档位预测: 1档(25.0%) | 2档(30.0%) | 3档(28.0%) | 4档(17.0%)
```

**档位预测特点**：
- **仅深度学习模式可用**：由LSTM模型输出
- **共享特征**：与类型预测共享Embedding+LSTM+共享FC层
- **独立输出**：通过档位预测头(4类)单独输出
- **多任务学习**：类型损失 + 档位损失联合优化

**档位数值含义**：
- 1档：最低数值（如暴击2.7%）
- 2档：较低数值（如暴击3.1%）
- 3档：较高数值（如暴击3.5%）
- 4档：最高数值（如暴击3.9%）

#### 置信度显示
显示预测的可靠程度，帮助判断预测结果的可信度：
```
置信度: 高 (45.2)  # 绿色 - 预测较可靠
置信度: 中 (28.5)  # 橙色 - 有一定参考价值
置信度: 低 (15.3)  # 红色 - 预测不确定，仅供参考
```

**置信度计算逻辑公式**：`置信度 = 最高概率 - 标准差/2`

> 注：`CONFIDENCE_STD_FACTOR` 配置已移除，固定使用 `/2` 计算

**原理**：
- 最高概率越高 → 基础置信度越高
- 标准差越大（分布越分散）→ 扣减越多
- 标准差越小（分布越集中）→ 扣减越少

**分级标准**（两种模式不同）：

| 等级 | 统计模式 | 深度学习模式 | 说明 |
|------|---------|-------------|------|
| 高 | ≥40 | ≥20 | 预测较可靠（绿色） |
| 中 | 25-40 | 15-20 | 有一定参考价值（橙色） |
| 低 | <25 | <15 | 仅供参考（红色） |

**为什么深度学习模式阈值更低？**
- 统计模式：查表得到，分布可能很集中（最高概率可达40%+）
- 深度学习：softmax输出，概率分布天然平滑（最高概率通常10-15%）

**示例**：
```python
概率分布: [44.4, 0, 0, 0, 4.4, 13.3, 4.4, 0, 16.7, 16.7]
最高概率: 44.4%
标准差: 16.8
置信度 = 44.4 - 16.8/2 = 36.0（中置信度）
```

### 5.4 统计模式与深度学习模式的区别

**统计模式**（已更新为滑动窗口）：
- **方法**：使用滑动窗口预测，最多利用 `config.MAX_ENTRIES_FOR_STATS` 个词条（默认7个）
- **机制**：生成多个3元组窗口，加权平均（近期窗口权重更高）
- **权重配置**：使用 `config.SLIDING_WINDOW_WEIGHTS` 配置（默认 [1,2,3,4,5,6,7]）
- **优势**：比单窗口更稳定，比纯滑动窗口更清晰

**深度学习模式**：
- **方法**：使用多skip策略，基于最近3-5个词条智能预测
- **原因**：LSTM有学习能力，可以从多个时间尺度学习模式
- **优势**：能捕捉近期、中期、长期多种模式，预测更全面

**对比**：
| 方面 | 统计模式 | 深度学习模式 |
|------|---------|-------------|
| 输入 | 最近7个词条（滑动窗口） | 最近3-5个词条（多skip加权） |
| 机制 | 滑动窗口加权平均 | 多skip策略（近期+中期+长期） |
| 学习能力 | 无（查表） | 有（LSTM推理） |
| 适用场景 | 通用预测 | 个性化预测（需训练） |

### 5.5 多重Skip策略的详细实现

**核心思想**：用不同时间尺度的3元组预测同一个目标

#### 预测逻辑

```
录入: [A, B, C, D, E, F, G] (7个词条，预测G后面的H)

skip=1 (近期):  [E, F, G] → 预测H (权重0.55)
skip=2 (中期):  [D, E, F] → 预测H (权重0.25)
skip=3 (长期):  [C, D, E] → 预测H (权重0.15)
skip=4 (超长期): [B, C, D] → 预测H (权重0.05)

结果 = 0.55×skip1 + 0.25×skip2 + 0.15×skip3 + 0.05×skip4
```

**关键设计**：
- 输入都是**连续的3个词条**，只是起始位置不同
- skip=1用最后3个，skip=2往前移1位，skip=3往前移2位
- 通过不同步长（1/2/3），都指向**同一个预测目标**

#### 词条数与skip选择

| 词条数 | 可用skip | 输入序列 |
|--------|---------|---------|
| 3 | 1 | [A,B,C] |
| 4 | 1, 2 | [B,C,D], [A,B,C] |
| 5-6 | 1, 2, 3 | [最后3个], [往前1位], [往前2位] |
| ≥7 | 1, 2, 3, 4 | [最后3个], [往前1位], [往前2位], [往前3位] |

#### 架构分层

```
artifact_predictor.py (业务逻辑层)
    └── predict_dl_smart()
        ├── 决定用哪些skip组合
        ├── 计算各skip权重
        └── 调用模型层预测

dl_model.py (模型层)
    └── predict()
        └── 单次LSTM推理
```

### 5.6 跨星级加权训练（3星垫刀→5星预测）

**背景**：3星垫刀和5星强化的词条生成规律可能不同，但跨星级样本很少。

**什么是跨星级样本？**
- 输入序列中同时包含3星和5星词条
- 或输入输出星级不同（如3星输入→5星输出）

**为什么需要加权？**
- 3星垫刀成本低，记录多（权重1.0）
- 5星强化成本高，记录少（权重1.0）
- 跨星级样本最稀缺但信息价值高（权重2.0）

**权重分配**：
| 样本类型 | 说明 | 权重 |
|---------|------|------|
| 纯3星 | 输入输出都是3星 | 1.0 |
| 纯5星 | 输入输出都是5星 | 1.0 |
| **跨星级** | 输入含3星+5星，或输入输出星级不同 | **2.0** |

**示例**：
```
纯3星: [f8(3星), s100(3星), g9(3星)] → 预测下一个类型（3星）权重1.0
纯5星: [F73(5星), c52(5星), B62(5星)] → 预测下一个类型（5星）权重1.0
跨星级: [b31(3星), c52(5星), F73(5星)] → 预测下一个类型（5星）权重2.0
         ↑ 输入同时含3星和5星
```

**检测逻辑**：
```python
is_transition = (has_3_star and has_5_star) or \
               (has_3_star and target_star == '5星') or \
               (has_5_star and target_star == '3星')
```

**加权损失函数**（使用 `reduction='none'` 方式）：
```python
# 初始化损失函数（关键：reduction='none' 获取每个样本的独立损失）
criterion = nn.CrossEntropyLoss(reduction='none')

# 使用 reduction='none' 获取每个样本的独立损失
loss = criterion(outputs, targets)  # shape: (batch_size,)

# 应用样本权重
weighted_loss = (loss * weights).mean()  # 跨星级样本权重×2

# 训练集：反向传播
weighted_loss.backward()

# 测试集：计算加权平均损失（用于早停判断）
test_loss = (loss * weights).mean().item()
```

**关键修复**：
- 使用 `reduction='none'` 确保每个样本损失独立计算
- 正确应用 `weights` 到每个样本
- 训练和测试都使用加权损失

## 六、关键概念解释

### 6.1 什么是过拟合？

**正常情况**：模型学习通用规律，能应对新数据  
**过拟合**：模型死记硬背训练数据，对新数据表现差

**例子：**
- 正常：学会"2+2=4"，遇到"3+3"也会算
- 过拟合：只记住"2+2=4"，遇到"3+3"就不会了

### 6.2 什么是Embedding？

**简单理解**：把数字变成向量，让模型理解词条间的关系

比如：
- f（小防）和 F（大防）都是防御，Embedding会让它们的向量比较接近
- f（小防）和 b（暴击）关系不大，向量距离较远

这样模型就能理解"防御词条倾向于连续出现"这种规律。

### 6.3 什么是LSTM？

**LSTM** = Long Short-Term Memory（长短期记忆网络）

**为什么用LSTM？**  
强化词条有先后顺序，LSTM能记住前面的信息：
- 记住第1条是什么
- 记住第2条是什么
- 记住第3条是什么
- 综合3条信息，预测第4条

普通神经网络记不住这么长的序列，LSTM专门解决"记忆"问题。

### 6.4 LSTM vs N-gram vs Transformer 对比

**三种序列模型的核心区别**：

| 模型 | 原理 | 记忆能力 | 数据需求 | 适用场景 |
|------|------|---------|---------|---------|
| **N-gram** | 统计频率 | ❌ 只记n个 | 极少 | 简单统计 |
| **LSTM** | 循环神经网络 | ⚠️ 会遗忘 | 中等(100+) | 中等长度序列 |
| **Transformer** | 注意力机制 | ✅ 优秀 | 大(1000+) | 长序列/复杂模式 |

**详细对比**：

```
序列: A → B → C → D → E → F → G → ?

N-gram(3元):
  只看 [E,F,G] → 预测?
  记不住A,B,C,D

LSTM:
  维护隐藏状态 h_t
  h_0→h_1→h_2→...→h_G
  理论上记住所有（实际会遗忘）

Transformer:
  直接看 [A,B,C,D,E,F,G] 全部
  注意力决定关注哪些
```
