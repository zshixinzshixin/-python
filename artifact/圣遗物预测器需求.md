# 原神圣遗物预测器需求

## 一、核心功能需求

### 1. 界面布局
- **左侧多标签页设计**：包含五星、三星、**胚子**三个标签页
  - 五星标签页：录入五星词条（显示具体数值如f16, f19等）
  - 三星标签页：录入三星词条（显示具体数值如f8, f9, f10, f11等）
  - 胚子标签页：选择4个目标词条，评估强化价值
- **中间录入区**：共用的录入区，可滚动显示无限个槽位
  - 最新词条在最上面：新录入的词条显示在列表顶部
  - 显示内容：档位、词条（具体数值）、星级（3星/5星）、录入顺序
- **右侧预测结果区**：共用的预测界面
  - 预测模式选择：统计模式/深度学习模式
  - 操作按钮：保存数据、导出Excel、训练模型
  - 状态显示：记录数、模型状态（已训练/未训练）
  - 表1：预测下一次词条（显示概率%）
  - 表2：胚子强化概率：显示4个目标词条的强化概率（%）

### 2. 词条输入
- **五星词条**：显示具体数值（f16, f19, f21, f23等），4个档位
- **三星词条**：显示具体数值（f8, f9, f10, f11; s100, s115, s129, s143等），4个档位
- **混合预测**：三星和五星的前三个词条共同预测结果
- **数据隔离**：左侧输入按钮的状态不共享，但中间录入区共用显示

### 3. 录入区功能
- **无限槽位**：可以滚动显示无限个录入槽位
- **最新词条置顶**：新录入的词条显示在最上面（第1条是最新的）
- **显示内容**：档位、词条（具体数值）、星级（3星/5星）、录入顺序（从1开始递增）
- **删除功能**：可以删除最后一个录入的词条
- **清空功能**：清空所有录入记录

### 4. 数据功能
- **保存数据**：将当前录入的词条保存为JSON（可随时保存）
  - 文件：`强化记录/record_YYYYMMDD_HHMMSS.json`
  - 内容：时间戳、所有词条详情（值、档位、星级、类型、顺序）
- **导出Excel**：将当前录入数据导出为Excel
  - 文件：`强化记录/exports/current_entries_YYYYMMDD_HHMMSS.xlsx`
  - 内容：录入顺序、词条值、档位、星级、类型
- **训练模型**：训练深度学习模型（需≥10条记录）

### 5. 预测功能
- **实时预测**：每输入一个词条就实时更新预测结果
- **预测逻辑**：
  - **统计模式**：使用滑动窗口预测，最多利用最近7个词条（多个3元组加权平均，近期窗口权重更高）
  - **深度学习模式**：使用多skip策略，基于最近3-5个词条智能预测（近期+中期+长期加权）
- **双模式预测**：
  - 统计模式：使用genmat401.npy查表预测
  - 深度学习模式：使用best_model.pt模型预测（需训练）
- **预测结果显示**：
  - **表1**：预测下一次词条 - 显示所有10种词条的概率%
  - **Top-3推荐**：显示概率最高的3个词条（蓝色）
  - **统计Top-3**（仅深度学习模式）：显示统计模式的Top-3推荐（黑色），用于双模式对比
  - **置信度**：显示预测可靠程度（高/中/低）
    - **计算公式**：`置信度 = 最高概率 - 标准差/2`
    - **逻辑**：最高概率越高、分布越集中，置信度越高
    - **分级标准**（两种模式不同）：
      - **统计模式**：≥40高（绿）、25-40中（橙）、<25低（红）
      - **深度学习模式**：≥20高（绿）、15-20中（橙）、<15低（红）
  - **档位预测**（仅深度学习模式）：
    - 基于LSTM模型预测下一个词条的档位（1-4档）概率
    - 与词条类型预测共享模型特征，同时输出
    - 高概率档位用颜色标注：
      - ≥35%：绿色（该档位出现概率较高）
      - ≥30%：黄色（该档位出现概率中等）
      - ≥25%：橙色（该档位出现概率略高）
      - <25%：白色（该档位出现概率较低）
    - 统计模式下不显示档位预测
  - **表2：胚子强化概率**（仅胚子标签页）：
    - 显示4个选定词条的条件概率（归一化为100%）
    - **强化建议**：基于双暴%给出操作建议
      - 双暴% = 暴击概率 + 暴伤概率
      - 歪率 = 100% - 双暴%（非bB词条总概率）
      - ≥60%：绿色（建议强化！）
      - 50-60%：橙色（可考虑强化）
      - <50%：红色（建议继续垫刀）
    - **风险度评估**：
      - 计算公式：`风险倍数 = max(非bB概率) / min(bB概率)`
      - <1.0x：绿色（安全，bB概率高于非bB）
      - 1.0-1.5x：橙色（警告，非bB概率略高）
      - >1.5x：红色（危险，非bB概率远高于bB）

## 二、使用流程

1. **选择标签页**：在左侧选择五星、三星或胚子标签页

## 四、预测算法说明

### 1. 统计模式

**模型数据结构（genmat401.npy）**：
- **S1/S2/S3**：三星词条统计矩阵
- **W1/W2/W3**：五星词条统计矩阵
- **Pcount**: 概率计数基准值

**NPY文件本质说明**：
NPY文件是**纯频率统计表**，不是N-gram推理结果：
- **S3矩阵形状**：(40, 40, 40, 10) = 输入3个词条索引 + 输出10种类型频率
- **数据内容**：`S3[i,j,k,t] = 次数`，表示词条i→j→k后，出现类型t的历史次数
- **覆盖率**：S3约78.87%的3元组组合有数据（基于大量3星垫刀记录）
- **预测方式**：直接查表归一化，无学习/推理过程

**词条索引映射**：
- **词条类型**：10种（f小防, s小生, g小攻, F大防, S大生, G大攻, j精通, c充能, b暴击, B暴伤）
- **档位数量**：每种类型4个档位
- **总索引数**：40 = 10种类型 × 4个档位
- **索引计算**：`索引 = 类型索引 × 4 + 档位索引`

**预测流程**（滑动窗口）：
- **最大词条数**：使用 `config.MAX_ENTRIES_FOR_STATS`（默认7个）
- **权重配置**：使用 `config.SLIDING_WINDOW_BASE`（默认0.3）和 `config.SLIDING_WINDOW_RANGE`（默认0.7）计算，公式：`weight = BASE + RANGE × 位置比例`
1. 获取最近7个词条（或所有词条如果不足7个）
2. 生成多个3元组窗口（如7个词条生成5个窗口）
3. 每个窗口查表得到概率分布
4. 加权平均：越近的窗口权重越高
5. 计算最终概率：`概率 = 加权平均(counts) / sum(counts) × 100%`

**滑动窗口机制**（已更新）：
- 从最近7个词条中生成多个3元组窗口
- 每个窗口查表得到概率分布
- 加权平均：越近的窗口权重越高
- 平衡了信息量和信号清晰度

**示例**（录入7个词条）：
```
窗口1: [词条1,2,3] 权重 0.07
窗口2: [词条2,3,4] 权重 0.13
窗口3: [词条3,4,5] 权重 0.20
窗口4: [词条4,5,6] 权重 0.27
窗口5: [词条5,6,7] 权重 0.33（最高）

最终概率 = 加权平均所有窗口结果
```

### 2. 深度学习模式

**词条编码**：
```
组合ID = 类型×4 + 档位 + 星级×40

例: f16(5星,档位1) = 0×4 + 0 + 1×40 = 40

总共 10类型 × 4档位 × 2星级 = 80种组合
```

**模型结构**：
```
输入(3个ID) → Embedding(16维) → LSTM(64×2层) → 全连接 → 输出(10类概率)
```

**训练条件**：≥10条记录

**多任务输出**：
- **类型预测**：10类概率分布（f,s,g,F,S,G,j,c,b,B）
- **档位预测**：4类概率分布（1档/2档/3档/4档）
- **共享特征**：Embedding + LSTM + 共享FC层
- **独立头**：类型预测头(10类) + 档位预测头(4类)

**预测机制（多skip策略）**：

深度学习模式**使用所有已录入的词条**：
- **训练时**：滑动窗口遍历所有历史数据，生成多skip训练样本
- **预测时**：多skip策略从不同时间尺度选取3元组进行预测

**核心理解**：预测时不同skip的输入序列都是**连续的3个词条**，只是**起始位置不同**，通过不同步长指向**同一预测目标**：

```
录入词条: [A, B, C, D, E, F, G] (7个，G是最新录入的，预测G后面那个)

预测时多skip选取：
skip=1 (近期): 用 [E, F, G] + skip=1 → 预测 G+1（走1步，权重0.55）
skip=2 (中期): 用 [D, E, F] + skip=2 → 预测 F+2 = G+1（走2步，权重0.25）
skip=3 (长期): 用 [C, D, E] + skip=3 → 预测 E+3 = G+1（走3步，权重0.15）
skip=4 (超长期): 用 [B, C, D] + skip=4 → 预测 D+4 = G+1（走4步，权重0.05）

注意：输入都是连续的3个词条，只是起始位置依次前移
      四个skip通过不同步长，都指向G后面的那个词条

最终结果 = 0.55×skip1 + 0.25×skip2 + 0.15×skip3 + 0.05×skip4
```

**skip配置**：
- **最大skip数**：`config.MAX_SKIP`（默认4，可配置1-5）
- **权重配置**：
  - `SKIP_1_WEIGHT_5TO6`：5-6词条时的skip=1权重（默认0.55）
  - `SKIP_1_WEIGHT_7PLUS`：≥7词条时的skip=1权重（默认0.55）
  - `SKIP_2_WEIGHT`：skip=2权重（默认0.25）
  - `SKIP_3_WEIGHT`：skip=3权重（默认0.15）
  - `SKIP_4_WEIGHT`：skip=4权重（默认0.05）

**词条数与skip选择**：
| 词条数 | 可用skip | 说明 |
|--------|---------|------|
| <3 | 无 | 无法预测 |
| 3 | 1 | 只有skip=1 |
| 4 | 1, 2 | 近期+中期 |
| 5-6 | 1, 2, 3 | 近期+中期+长期 |
| ≥7 | 1, 2, 3, 4 | 四者综合 |

**模型兼容性**：
- 旧模型（max_skip=3）与 skip=4 配置兼容
- 预测时自动检测模型支持的 max_skip，取配置和模型的最小值

### 3. 胚子评估算法

**条件概率计算**：
```
P(X | 胚子词条) = P(X) / P(胚子总概率)

其中：
- P(X)：词条X的原始预测概率
- P(胚子总概率)：4个胚子词条概率之和
- 结果归一化：4个条件概率之和 = 100%
```

**强化建议逻辑**：
```
双暴% = P(暴击) + P(暴伤)
歪率 = 100% - 双暴%

建议规则：
- 双暴% ≥ 60% → 绿色（建议强化！）
- 50% ≤ 双暴% < 60% → 橙色（可考虑强化）
- 双暴% < 50% → 红色（建议继续垫刀）
```

**风险度评估**：
```
风险倍数 = max(非bB概率) / min(bB概率)

风险等级：
- < 1.0x → 绿色（安全）：bB概率高于所有非bB
- 1.0x - 1.5x → 橙色（警告）：最高非bB略高于最低bB
- > 1.5x → 红色（危险）：最高非bB远高于最低bB

示例：
胚子：暴击(30%) + 暴伤(30%) + 精通(35%) + 充能(5%)
风险倍数 = 35 / 30 = 1.17x → 橙色（警告）
```

## 五、常见问题

**Q: 为什么深度学习模式是灰色的？**  
A: 需要模型文件存在（best_model.pt）

**Q: 保存会覆盖之前的数据吗？**  
A: 不会，每次保存生成新文件（带时间戳）

**Q: PT模型和NPY文件有什么区别？**  
A: NPY是通用统计模型，PT是个人数据训练的深度学习模型

**Q: 训练需要多久？**
A: 10条约10-20秒，100条约1-2分钟，500条约3-5分钟
